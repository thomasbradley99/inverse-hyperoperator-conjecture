# Formal Framework (Proof-Track Draft)

This note upgrades the current roadmap into a precise theorem framework with
explicit assumptions, objects, and proof obligations.

It is written to make collaboration easier: each lemma can be attacked
independently.

## 1) Objects and notation

- Forward map: `F(z) = z^z = exp(z Log z)` on a chosen branch of `Log`.
- Value domain: `U` in the `w`-plane where inverse continuation is studied.
- Branch set (inverse singular values / excluded set): `Sigma`.
- Working domain: `U = C \ Sigma` (or a connected component of that complement).
- Based point: `w0 in U`.
- Local inverse germ at `w0`: `g0`.
- Analytic continuation along a based loop `gamma` gives another inverse germ.

Monodromy action:

- `rho: pi_1(U, w0) -> Perm(B)` where `B` is the set of inverse branches over `w0`.
- Deck/branch shift image `G = im(rho)`.

Computationally, branch labels are represented by integer pairs `(k,m)` via
`exp(W_m(log w + 2*pi*i*k))`, but the theorem track should avoid assuming this
formula globally and instead use continuation/covering structure.

## 2) Near-term theorem statement (formalized)

Assume:

1. `U` is connected and locally path-connected.
2. Inverse branches of `F` define a covering-like continuation structure over `U`.
3. The induced branch/deck action contains a free abelian rank-2 subgroup
   generated by two independent loop actions.

Then:

- (T1) No globally continuous single-valued map `s: U -> C` can realize all
  inverse continuations as one branch selection.
- (T2) Any faithful global continuous state encoding that resolves branch
  ambiguity needs at least two extra real coordinates beyond `(Re z, Im z)`.
  Equivalently, any such encoding has real dimension at least `4`.

`T2` is the rigorous version of "rank 2 branch data forces +2 real dimensions."

## 3) Proof obligations (checklist)

### A. Domain and continuation setup

- A1. Specify admissible `U` and excluded `Sigma` precisely.
- A2. Prove continuation of local inverse branches is well-defined along paths in `U`.
- A3. Prove branch transport depends only on homotopy class of based loops.

### B. Monodromy/deck structure

- B1. Exhibit one loop with nontrivial branch action.
- B2. Exhibit two loops with independent actions.
- B3. Prove the generated subgroup has rank at least `2` (or provide equivalent criterion).

### C. Global-obstruction statement

- C1. Show nontrivial monodromy obstructs single-valued global continuous branch choice.
- C2. Formalize "faithful encoding" of branch state.
- C3. Prove faithful encoding must inject at least two independent branch parameters.
- C4. Conclude lower bound `dim_R >= 2 + 2 = 4`.

## 4) How current numerics map to obligations

- Scripts `01`-`04`: evidence for A/B/C intuition (gap, shifts, deck closure).
- Script `05`: composition/winding linearity and cancellation checks.
- Script `06`: tracker robustness cross-check.
- Script `07`: based-loop order sensitivity and commutator diagnostics.

These do **not** replace proofs of A1-A3/B1-B3/C1-C4; they guide which formal
claims are plausible and where subtlety lives (especially non-abelian loop effects).

## 5) Caution points for the proof

- Avoid inferring full group structure from finite sampled loops.
- Distinguish:
  - abelianized winding data,
  - based-loop class in `pi_1`,
  - actual monodromy representation image.
- Keep theorem scope local to a clearly defined domain `U` first.
- Separate dimension lower bound theorem from the stronger quaternion-uniqueness claim.

## 6) Deliverable sequence (recommended)

1. Prove T1 on a fixed explicit domain `U`.
2. Prove T2 (dimension lower bound) under a clean "faithful encoding" definition.
3. Only then attack structural uniqueness (`H` vs generic `R^4` model).

This sequence turns the project from conjectural narrative into a staged theorem program.
